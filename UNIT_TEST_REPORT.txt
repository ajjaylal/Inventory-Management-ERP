[FAIL]  UI-08: Stock "Add Stock Item" — UOM dropdown stuck on
               "Loading units..." in some browser sessions
        Result: The Unit of Measure dropdown renders a loading state and
                never populates options. The fetch from Supabase is silently
                failing with no visible error to the user.
        Root cause: The `fetchData()` function in `stock/page.tsx` fetches
                    UOMs and sets them in state. If the Supabase anon-key
                    session cookie is not properly attached (can happen after
                    dev-server restart without refreshing the login), the
                    fetch returns empty data with no thrown error.
        Impact: HIGH — Users cannot add any stock items at all.
        How to fix:
          1. In `stock/page.tsx` line 58-62, change the UOM fetch to log
             errors explicitly:
               const { data: uomData, error: uomErr } = await supabase
                 .from('units_of_measure')
                 .select('*')
                 .order('name');
               if (uomErr) console.error('UOM fetch failed:', uomErr);
               if (uomData) setUoms(uomData);
          2. Open browser DevTools > Console to see the exact Supabase error
             when the dropdown is stuck.
          3. If the error is "JWT expired" or auth-related, the fix is to
             force the user to log out and log back in, or implement a
             session refresh. The `src/lib/supabase/middleware.ts`
             `updateSession()` function handles token refresh — ensure it
             is being called before each page load.
          4. If this consistently fails after a fresh login, check that
             NEXT_PUBLIC_SUPABASE_ANON_KEY in .env.local is correct and
             has not expired.
